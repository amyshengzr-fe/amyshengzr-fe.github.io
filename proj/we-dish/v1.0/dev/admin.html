We Dish Admin — 开发指南 ​

技术栈 ​

  • 框架：Vue 3 + Vite 7
  • 路由管理：Vue Router 4.x
  • UI 库：Element Plus 2.x
  • 状态管理：Pinia 3.x
  • HTTP：axios
  • 语言：TypeScript 5.x
  • 样式：Unocss 66.x
  • 工具库：js-cookie（鉴权）、VueUse 14

请求封装（src/lib/request.ts） ​

该文件为 HTTP 请求的核心封装，处理：

  • Token 注入：从 cookie 中读取 VITE_API_KEY，自动在 Authorization 头注入
  • 错误处理：401 时清理 token 并跳转登录，其他错误通过 message 提示
  • 超时配置：默认 10s，chat 接口可覆盖为 60s
  • Content-Type：默认 multipart/form-data，JSON 请求需显式覆盖

1. 环境变量与配置 ​

.env/.env.local 中定义 API 前缀和 token cookie key，在 lib/request/ 中读取：

const baseURL = import.meta.env.VITE_API_PREFIX;

2. HTTP 请求与鉴权 ​

typescript
import axios from "axios";
import { ElMessage } from "element-plus";
import cookie from "js-cookie";

const key = import.meta.env.VITE_API_KEY;

const request = axios.create({
  baseURL: import.meta.env.VITE_API_PREFIX,
  timeout: 10000,
  headers: {
    "Content-Type": "multipart/form-data",
  },
});

request.interceptors.request.use(function (config) {
  const token = cookie.get(key);
  (config as any).headers = {
    ...config.headers,
    Authorization: !!token ? token : "",
  };
  return config;
});

request.interceptors.response.use(function (response) {
  const data = response?.data;

  if (data?.code === 401) {
    cookie.remove(key);
    ElMessage.error(data?.msg);
    if (location.pathname !== "/login") {
      setTimeout(() => {
        location.replace("/login");
      }, 200);
    }
  } else if (data?.code !== 0) {
    ElMessage.error(data?.msg);
  }
  return response;
});

export default request;

3. Pinia 状态管理 ​

typescript
// stores/userSlice.ts
import { computed, ref } from "vue";
import { defineSetupStore } from "../helper";
import type { User } from "@/api/user/user.types";
import cookie from "js-cookie";
import { logout } from "@/api/session/session";
import { ElMessage } from "element-plus";
import { getUserInfo } from "@/api/user/user";
import { resetAllStores } from "../reset";

export const useUserStore = defineSetupStore("user", () => {
  const info = ref<User>({
    account: "",
    dishes_count: 0,
    id: 0,
    nickname: "",
    avatar: "",
    email: "",
  });
  const loaded = ref(false);
  const isLoggedIn = computed(() => !!cookie.get(import.meta.env.VITE_API_KEY));
  const isMobile = ref(false);

  const setUserInfo = (data: User) => (info.value = data);

  const setIsMobile = (value: boolean) => (isMobile.value = value);

  const fetchUserInfo = async () => {
    if (loaded.value) return;
    const res = await getUserInfo();
    if (res?.code === 0) {
      setUserInfo(res.data);
      loaded.value = true;
    }
  };

  const handleLogout = async () => {
    const res = await logout();
    if (res?.code === 0) {
      cookie.remove(import.meta.env.VITE_API_KEY);
      ElMessage.success("退出登录成功");
      resetAllStores();
      location.href = "/login";
    }
  };

  return {
    info,
    isMobile,
    isLoggedIn,
    setUserInfo,
    setIsMobile,
    fetchUserInfo,
    logout: handleLogout,
  };
});

4. 权限控制 ​

在路由守卫中检查用户权限：

typescript
// router/index.ts
router.beforeEach((to, from, next) => {
  const userStore = useUserStore();
  const useRoute = useRouteStore();
  // document.title = (to.meta.title as string) || 'We-Dish'
  const isLoggedIn = userStore.isLoggedIn;

  if (isLoggedIn && to.name !== "login" && to.name !== "notfound") {
    useRoute.addRecord(to.path, to.meta.title, to.name);
  }

  if (!isLoggedIn && to.meta.requiresAuth) {
    next({ name: "login", query: { redirect: to.fullPath } });
  } else if (to.name === "login" && isLoggedIn) {
    next({ name: "home" });
  } else {
    next();
  }
});

性能优化 ​

1. 构建性能 ​

  • 使用 Vite 的依赖预构建优化（optimizeDeps）

    typescript
    // 明确指定预构建依赖，减少首次启动和热更新成本；
// 特别是 Element Plus、axios 等较大的依赖。
 optimizeDeps: {
  include: [
    'element-plus',
    '@element-plus/icons-vue',
    'axios',
    'vue',
    'vue-router',
    '@vueuse/core',
    'pinia',
  ],
},

  • 使用 Vite 的生产构建优化（build）

    typescript
      // 调整 chunk 大小拆分策略，减少单文件体积。
  build: {
    chunkSizeWarningLimit: 800,
    rollupOptions: {
      // 把经常不变的大库单独打包，便于浏览器长期缓存；
      output: {
        manualChunks: {
          vue: ['vue', 'vue-router', 'pinia'],
          elementPlus: ['element-plus'],
          axios: ['axios'],
        },
      },
    },
},

  • 使用 Vite 插件提升构建性能

    typescript
    import { type PluginOption } from "vite";
import viteCompression from "vite-plugin-compression";
import { visualizer } from "rollup-plugin-visualizer";
import Inspect from "vite-plugin-inspect";

const isProd = process.env.NODE_ENV === 'production';

pllugins: [
  // 生成 gzip / brotli 压缩后的静态文件,让最终构建体积更小，提升首屏加载速度
  isProd &&
    viteCompression({
      algorithm: 'gzip',
      ext: '.gz',
      threshold: 10240, // 10KB 以上文件才压缩
      deleteOriginFile: false,
    }),
  isProd &&
    viteCompression({
      algorithm: 'brotliCompress',
      ext: '.br',
      threshold: 10240, // 10KB 以上文件才压缩
      deleteOriginFile: false,
    }),
  // 用于分析打包体积、查找大依赖、决定如何拆 chunk。(在 build 后生成 stats.html)
  visualizer({
    filename: './dist/stats.html', // 输出文件
    open: true, // 构建完成后自动打开
    gzipSize: true, // 显示 gzip 体积
    brotliSize: true, // 显示 brotli 体积
  }) as PluginOption,
  // 开发时用于调试插件和模块处理流程
  Inspect(),
].filter(Boolean), //isProd && compression() 若为 false 会成为 false，最后被过滤掉

2. 运行时性能 ​

  • 组件级 Lazy Loading

    typescript
      i. 路由页面使用 () => import() 已默认懒加载；
  ii. 组件中也可按需加载：
      const AddEdit = defineAsyncComponent(() => import('@/components/Dish/AddEdit.vue'))

  • 避免不必要的 reactive / ref 包装, 尽量使用 shallowRef、shallowReactive 处理大数据（表格数据、图表数据）。

  • 减少无必要的 Computed、Watch 开销

  • 避免在模板内执行重计算,如不要直接在 template 写 items.filter(...)。

  • Element Plus 按需加载

    typescript
    import AutoImport from 'unplugin-auto-import/vite'
import Components from 'unplugin-vue-components/vite'
import { ElementPlusResolver } from 'unplugin-vue-components/resolvers'

plugins: [
  AutoImport({
    resolvers: [ElementPlusResolver()],

    resolvers: [ElementPlusResolver()],

3. 状态管理性能（Pinia） ​

  • 拆分 Store，减少全局 store 体积, 避免一个 store 过大导致订阅过多

  • 对不需要响应式的数据使用非响应式存储

    typescript
    i. 使用 markRaw()；
ii. 使用localStorage / sessionStorage；
iii. 避免 Pinia 过度响应化。

4. 网络 & HTTP 优优化（axios） ​

  • 请求合并（axios + debounce）,如搜索框输入请求，使用 debounce 限流。
  • 服务端分页 & 懒加载
  • axios 按环境拆分基础配置
