接口规范 ​

本文件为 v1.0 版本的接口规范，适用于we-dish-lite（用户版）与 we-dish-admin（管理版） ,内容来源于项目内前端实现与已有文档的汇总。

1. 全局约定 ​

  • 统一响应格式

    json
    {
  "code": 0,
  "msg": "success",
  "data": {}
}

# code === 0：成功
# code === 401：鉴权失败（前端需清理 token 并跳转登录）
# 其他非 0：业务错误，msg 包含具体信息

  • Token 认证：请求在 Authorization 头携带 token

  • 超时与 Content-Type

    • timeout 默认超时：10s, /chat/ai 接口建议：60s（请求级别覆盖）
    • 因为后端的原因默认设置 Content-Type 为：multipart/form-data, 创建订单和 AI 推荐接口设置 Content-Type 为：application/json

2. 数据模型 ​

2.1 User ​

typescript
export interface User {
  account: string; // 账号
  avatar?: string; // 头像
  dishes_count: number; // 菜品数量
  id: number; // 用户 ID
  nickname: string; // 昵称
}

2.2 Dish ​

typescript
export interface Dish {
  create_time?: string; // 创建时间
  desc?: string; // 菜品描述
  id?: number; // 菜品 ID
  name: string; // 菜品名称
  picture?: string; // 菜品图片
  tags: Tags; // 菜品标签
  type: string; // 菜品类型
  update_time?: string; // 更新时间
}

export interface Tags {
  cuisine: string[]; // 菜系
  not_recommended_for: string[]; // 不推荐人群
  recommended_for: string[]; // 推荐人群
}

2.3 Order ​

typescript
export interface Order {
  create_time: string; // 创建时间
  dish: OrderDish[]; // 订单菜品列表
  id: number; // 订单 ID
  order_code: string; // 订单编号
}

export interface OrderDish {
  extras?: string[] | null; // 额外信息：葱、蒜、香菜等 (ONION: "加葱", GINGER: "加姜",GARLIC: "加蒜",CILANTRO: "加香菜")
  id: number; // 订单菜品 ID
  name: string; // 菜品名称
  picture: string; // 菜品图片
  quantity: number; // 菜品数量
  spiciness: string; // 辣度 ( NO_SPICY: "不辣",TINY_SPICY: "微微辣",MEDIUM_SPICY: "微辣",SLIGHTLY_SPICY: "中辣",HOT_SPICY: "重辣")
  type: string; // 菜品类型
  cuisine: string; // 菜系
  desc: string; // 菜品描述
  not_recommended_for: string; // 不推荐人群
  recommended_for: string; // 推荐人群
}

3. 关键接口 ​

3.1 账号（Session） ​

  • POST /session/register - 用户注册

    typescript
    //请求
{
  account: string; // 账号
  password: string; // 密码
}

//响应
{
  code: number;
  msg: string;
}

  • POST /session/login - 用户登录，返回 token

    typescript
    //请求
{
  account: string; // 账号
  password: string; // 密码
}

//响应
{
  code: number;
  msg: string;
  data: {
    token: string; // token
    token_type: string; // token 类型
    user: User; // 用户信息
  }
}

  • POST /session/logout - 用户登出

    typescript
    //响应
{
  code: number;
  msg: string;
}
