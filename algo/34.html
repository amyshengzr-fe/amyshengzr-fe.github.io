34. 在排序数组中查找元素的第一个和最后一个位置 ​

题目描述 ​

给你一个按照非递减顺序排列的整数数组 nums，和一个目标值 target。请你找出给定目标值在数组中的开始位置和结束位置。

如果数组中不存在目标值 target，返回 [-1, -1]。

你必须设计并实现时间复杂度为 O(log n) 的算法解决此问题。

示例 1：

输入：nums = [5,7,7,8,8,10], target = 8
输出：[3,4]

示例 2：

输入：nums = [5,7,7,8,8,10], target = 6
输出：[-1,-1]

示例 3：

输入：nums = [], target = 0
输出：[-1,-1]

提示：

  • 0 <= nums.length <= 10⁵
  • -10⁹ <= nums[i] <= 10⁹
  • nums 是一个非递减数组
  • -10⁹ <= target <= 10⁹

题解 ​

代码 ​

javascript
/**
 * @param {number[]} nums
 * @param {number} target
 * @return {number[]}
 */

var searchRange = function (nums, target) {
  let start = -1,
    end = -1;
  let left = 0,
    right = nums.length - 1;

  while (left <= right) {
    let mid = (left + right) >> 1;
    if (nums[mid] >= target) {
      right = mid - 1;
      if (nums[mid] === target) {
        start = mid;
      }
    } else {
      left = mid + 1;
    }
  }

  (left = 0), (right = nums.length - 1);

  while (left <= right) {
    let mid = (left + right) >> 1;
    if (nums[mid] <= target) {
      left = mid + 1;
      if (nums[mid] === target) {
        end = mid;
      }
    } else {
      right = mid - 1;
    }
  }

  return [start, end];
};

实现逻辑 ​

使用两次二分查找

  • 第一次：查找左边界
  • 第二次：查找右边界

第一次二分：查找起始位置（左边界） ​

核心逻辑解释 ​

目标：尽可能向左逼近 target

  • 若 nums[mid] >= target

    • 即使等于 target，也不能停
    • 必须继续向左找
    • 所以 right = mid - 1

  • 若 nums[mid] === target

    • 记录当前位置为候选答案 start
    • 继续向左压缩区间

关键思想 ​

找到一个 target 并不停止，而是继续往左找是否还有更靠前的 target

最终：

  • start 会被更新为 最左边的 target 下标
  • 若不存在，则保持 -1

核心逻辑解释 ​

目标：尽可能向右逼近 target

  • 若 nums[mid] <= target

    • 即使等于 target，也要继续向右
    • 所以 left = mid + 1

  • 若 nums[mid] === target

    • 记录当前位置为候选答案 end

关键思想 ​

找到 target 后继续向右搜索，确保这是最后一次出现的位置

最终：

  • end 为最右侧 target 的下标
  • 若不存在，保持 -1

时间与空间复杂度 ​

  • 时间复杂度：

    • 两次二分查找
    • O(log n)

  • 空间复杂度：

    • 仅使用常量变量
    • O(1)

总结 ​

  • 本题利用数组有序的特性，通过两次二分查找分别确定目标值的左边界和右边界。
  • 第一次二分在遇到等于目标值时继续向左搜索，确保找到最小下标；
  • 第二次二分在遇到等于目标值时继续向右搜索，确保找到最大下标。
  • 若目标值不存在，则对应边界保持为 -1。
  • 整体时间复杂度为 O(log n)，空间复杂度为 O(1)。
